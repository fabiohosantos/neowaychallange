    <article class="markdown-body entry-content p-5" itemprop="text"><h1><a id="user-content-exercício" class="anchor" aria-hidden="true" href="#exercício"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Exercício</h1>
<h2><a id="user-content-objetivo" class="anchor" aria-hidden="true" href="#objetivo"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Objetivo</h2>
<p>O objetivo desse exercício é nos ajudar a conhecer melhor as suas habilidades de desenvolvimento, bem
como lhe apresentar um pouco dos desafios que você enfrentará no dia a dia na missão de ajudar o time
de analytics ser mais produtivo.</p>
<h2><a id="user-content-o-problema" class="anchor" aria-hidden="true" href="#o-problema"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>O Problema</h2>
<p>O time de product data science desenvolveu dezenas de modelos que rodam mensalmente em batch, o processo de
classificação é disparado utilizando um scheduler, no caso o <a href="https://airflow.apache.org" rel="nofollow">apache airflow</a>.</p>
<p>Cada classificador de modelo (job) é responsável em acessar variadas fontes de dados, executar a classificação
de fato, e gerar arquivos json no sistema de arquivos do scheduler. A convenção <a href="http://jsonlines.org" rel="nofollow">jsonlines</a>
é utilizada em todos os projetos. Um exemplo desse arquivo é:</p>
<div class="highlight highlight-source-json"><pre>{ <span class="pl-s"><span class="pl-pds">"</span>pk<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>123<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>score<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>100<span class="pl-pds">"</span></span> }
{ <span class="pl-s"><span class="pl-pds">"</span>pk<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>321<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>score<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>1<span class="pl-pds">"</span></span> }
{ <span class="pl-s"><span class="pl-pds">"</span>pk<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>50<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>score<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>98<span class="pl-pds">"</span></span> }</pre></div>
<p>Para que o resultado do esforço dos data sciences ganhe valor, esses dados devem ser injetados na plataforma
para que outras aplicações façam uso desses indicadores. Isso é realizado inserindo esses dados num banco
de dados <a href="https://www.postgresql.org" rel="nofollow">postgresql</a>.</p>
<p>Mas essa banco de dados é mantido por outro time, com muito carinho e esmero, e em um ambiente extremamente
seguro, e não é possível realizar conexão diretamente com esse banco.</p>
<p>Para acessar esse banco de dados é necessário um serviço http, que executará em um ambiente seguro e monitorado
pelo time de <strong>SRE</strong>.</p>
<h2><a id="user-content-a-missão" class="anchor" aria-hidden="true" href="#a-missão"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>A Missão</h2>
<ul>
<li>
<p>Desenvolver o serviço http que receberá esses dados do modelo e efetuará a inseção no postgresql.</p>
</li>
<li>
<p>Desenvolver um cliente pra esse serviço, que receba como entrada um diretório contendo arquivos com os dados
e envie para o serviço http.</p>
</li>
</ul>
<h2><a id="user-content-considerações" class="anchor" aria-hidden="true" href="#considerações"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Considerações</h2>
<p>Atente para algumas considerações importantes:</p>
<ul>
<li>
<p>Não existe resposta correta, o que procuramos é entender a sua capacidade de solucionar o problema e sua criatividade</p>
</li>
<li>
<p>A linguagem de programação comumente utilizada pelo time é <a href="https://www.python.org" rel="nofollow">python3</a>, utilizar python
vai facilitar a manutenção do software, mas se você for mais versado em outra linguagem, fique à vontade.</p>
</li>
<li>
<p>O banco de dados também é ao seu gosto, mas ter schema é importante.</p>
</li>
<li>
<p><a href="https://www.docker.com/" rel="nofollow">Docker</a> é largamente utilizado e gostaríamos muito de ver as suas habilidades com essa
ferramenta.</p>
</li>
<li>
<p>Testar é legal!</p>
</li>
<li>
<p>Mostre-nos também tudo o que você sabe sobre boas práticas de <a href="https://git-scm.com" rel="nofollow">git</a></p>
</li>
<li>
<p>Pense sempre na qualidade do código, tenha a manutenibilidade, legibilidade e simplicidade em mente e lute contra
<a href="https://en.wikipedia.org/wiki/Overengineering" rel="nofollow">overengineering</a>.</p>
</li>
<li>
<p>Lembre que são vários modelos, e cada modelo gera uma grande volume de dados, temos um recorde de 4,5 GB.</p>
</li>
<li>
<p>É desejável que consigamos executar o seu trabalho em laboratório.</p>
</li>
</ul>
